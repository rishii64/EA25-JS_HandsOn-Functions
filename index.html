Q4.Print output: 
                var x=21
                var girl = function(){
                    console.log(x)
                    var x=20
                }
                girl();
             
                 Output:
                 undefined

                 Reason:
                 This is due to variable hosting, i.e. even though we have declared x inside the girl () function,
                 the declaration itself is hoisted at the top of the function. Declared with var keyword, on trying
                 to access the variable before initialisation gives undefined as output.
                 Secondly, inside the girl() function the variable x has a local scope which shadows the outer x 
                 within the function scope.
Q5.Print output: 
 
                var x = 21;
                girl ();
                console.log(x)
                function girl() {
                    console.log(x);
                    var x = 20;
                };
             
                 Output:
                 undefined
                 21

                 Reason:
                 Like the previous question, js engine iterates the code line by line and the girl() function gets
                 called first where hoising and shadowing runs simultanesously giving the output as "undefined".
                 After that it prints 21.
Q6.Print output 
                var x = 21;
                a();
                b();
                function a() {
                x = 20;
                console.log(x);
                };
                function b() {
                    x = 40;
                console.log(x);
                };
             
                Output:
                20
                40

                Reason
                Js code executes line by line. First the a() function is called, where finding for x the JS compiler 
                goes to the previous line i.e. the global scope. Getting the value of x it consoles 
                where it gets changed to 20. 
                
                In the second output, same scenario occurs. Only the difference is that the value of x 
                changes to 40.
Q7. Write a function that accepts parameter n and returns factorial of n. 
                Answer:
                function fact(n){
                    let fact=1
                    for(let i=1;i<=n;i++)
                        fact *=i
                    return `Factorial of ${n} = ${fact}`;
                }
                console.log(fact(5))
             
                Output:
                Factorial of 5 = 120

Q1.Guess the Output 
                    function FindSum(a, b){
                        return a + b;
                    }
                    function DisplayData(data, batch){
                        console.log(`i am from ${data} and My batch is EA${batch}`)
                    }
                    DisplayData("PrepBytes", FindSum(10, 9));

Q3.Guess the Output

                        console.log(a);
                        var a = 20;

                    undefined

                    Reason:
                    Variable hoisting in javascript. Accessing the variable declared with var keyword before it's initialisation.

Q4.Guess the Output 
                    const greet =  function(name){
                        return function(m){
                            console.log(`Hi!! ${name}, ${m}`);
                        }
                    }
                    const greet_message = greet('EA19');
                    greet_message("Welcome To PrepBytes")
                
                    Output:
                    Hi!! EA19, Welcome To PrepBytes

                    Reason:
                    First, we call the function greet with an argument "EA19" at line #9 and stores the returned 
                    value in a variable called greet_message. After executing greet(), greet_message also becomes 
                    a function because we return a function from greet().
                    Secondly, we call the function  greet_message with an argument at line #10. After the execution 
                    it prints -  "Hi!! EA19, Welcome to Prepbytes". 
                    In the given program, the concept of closures comes into action.
